오라클과 SQLite에 대해 : 

SQLite DB와 Oracle은 서로 다른 데이터베이스 관리 시스템(DBMS)입니다. 
 SQLite는 경량의 서버리스 데이터베이스 엔진으로, 파일 기반의 데이터베이스를 제공하며, 주로 개인용 또는 작은 규모의 애플리케이션에서 사용됩니다. 

반면에 Oracle은 대규모 기업 및 기관에서 사용되는 강력한 관계형 데이터베이스 관리 시스템입니다.

 SQLite는 오픈 소스이며 퍼블릭 도메인에 있는 퍼미시브 라이선스를 따르므로 무료로 사용할 수 있습니다. 

반면에 Oracle은 상용 제품으로, 사용 및 라이선싱에 대한 비용이 발생할 수 있습니다.

SQLite와 Oracle은 SQL(Structured Query Language)을 사용하여 데이터를 관리하고 쿼리를 실행하는 데 공통적으로 사용될 수 있지만, 그들은 다른 특성과 용도를 갖고 있습니다. SQLite는 단일 파일에 저장되고 로컬에서 실행되며 경량화되어 있습니다. Oracle은 클라이언트-서버 아키텍처를 사용하며, 대규모의 데이터 처리와 고급 기능을 제공합니다.

따라서 SQLite DB와 Oracle은 서로 다른 시나리오에서 사용되는 다른 종류의 데이터베이스 시스템입니다.



>> Flask 연동/ 명령 프롬프트에서 쓰기

pip를 사용하여 Flask 설치: 명령 프롬프트에서 다음 명령을 실행하여 Flask를 설치합니다.

∴ pip install Flask

PATH 설정: Flask가 설치된 디렉터리를 시스템의 PATH 환경 변수에 추가합니다. Flask를 설치한 위치에 따라 경로가 달라질 수 있으므로, Flask가 설치된 경로를 찾고 시스템의 환경 변수에 추가해야 합니다.

설치가 완료되면 명령 프롬프트에서 flask run을 실행하여 Flask 애플리케이션을 실행할 수 있어야 합니다. 만약 PATH에 Flask를 추가하지 않았다면, python -m flask run 명령을 사용하여 Flask를 실행할 수도 있습니다.
